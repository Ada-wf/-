## 二、Phase 1: MVP（第 1-2 周，7/7 ~ 7/20）

### 目标
一个可部署、可浏览的职业百科网站，含 50 个职业、12 门类覆盖、首页卡片浏览、分类导航、思维导图、搜索和职业详情页。

### Week 1（7/7 ~ 7/13）: 基础搭建 + 内容生产

| 日期 | 任务 | 产出 | 完成标准 |
|------|------|------|---------|
| Day 1 (7/7) | 项目脚手架搭建 | Next.js 15 + Tailwind + TypeScript + Velite 初始化 `yarn dev` 可运行 | `localhost:3000` 显示 Hello World |
| Day 2-3 (7/8-9) | 数据模型实现 | `src/types/career.ts` + `src/types/category.ts` + `velite.config.ts` 配置完成 | TypeScript 编译通过；Velite 可读取 .md |

## 三、Phase 2: AI 动态补全（第 3 周，7/21 ~ 7/27）

### 目标
搜索未命中的职业可以自动联网搜索 → AI 生成 → 审核 → 图片生成 → 保存到站点。

### 任务拆解

| 日期 | 任务 | 产出 | 完成标准 |
|------|------|------|---------|
| Day 1 (7/21) | Tavily Search 集成 | `src/lib/search-web.ts` | 搜索返回 3-5 个有效结果；API 异常时返回空 |
| Day 1-2 (7/21-22) | Claude 文本生成 | `src/lib/ai-generate.ts` 核心函数 | 完整 Prompt 模板 (含分类树 + Few-shot)；JSON Schema 约束输出；Zod 解析成功 |
| Day 2-3 (7/22-23) | 内容审核 Step | `src/lib/ai-review.ts` | 审核通过/不通过判定；severity 分级处理 |
| Day 3 (7/23) | DALL·E + Replicate 集成 | `src/lib/image-generate.ts` | DALL·E 可用时优先；不可用时自动切换 Replicate；英文 prompt 翻译 |
| Day 3-4 (7/23-24) | Cloudinary 上传 + Vercel KV 存储 + 审计日志 | 图片 CDN + 数据持久化 + 审计 | 上传成功；KV 可读写；审计 key 正确记录 |
| Day 4-5 (7/24-25) | SSE 流式响应 | `POST /api/ai-generate` 返回 SSE | 7 种事件类型正确推送；前端实时显示进度 |
| Day 5-6 (7/25-26) | 前端 AI 面板 + 免责声明 | `AIGeneratePanel.tsx` + `AIDisclaimer.tsx` | 进度条 + 状态切换 + 完成展示 + 免责标记 + 反馈按钮 |
| Day 6-7 (7/26-27) | 用户反馈功能 | `POST /api/feedback` + 前端 👍👎 按钮 | 反馈可提交；重复投票拦截 |
| Day 7 (7/27) | 端到端测试 + Bug 修复 | 手动走通完整 AI 生成流程 3 次 | 3 次均成功生成并持久化 |

## 六、关键风险与应对

| 风险 | 信号 | 应对 |
|------|------|------|
| **职业内容生产速度慢** | Day 5 时只完成 < 10 个 | 优先高需求职业；用 AI 辅助生成初稿再人工修改 |
| **G6 四级树图性能问题** | 展开 50+ 节点后卡顿 | 默认只展开前两级；懒加载子节点；虚拟化渲染 |
| **AI 生成质量不稳定** | 3 次测试中 > 1 次审核不通过 | 调整 Prompt；收紧搜索源筛选；增加 few-shot 数量 |
| **API 费用超预期** | 月费用 > $150 | 降低单用户 AI 次数限制；启用图片缓存；切换到 Replicate 为主 |
| **部署/编译错误** | CI 红灯 | 优先修复 blocking 问题；非阻塞问题记录后 Phase 3 解决 |

## 七、日常开发流程

```
每日流程:
  09:00 - 10:00  站会 + Code Review 昨天的 PR
  10:00 - 12:00  编码 (核心逻辑)
  12:00 - 14:00  午饭 + 休息
  14:00 - 17:00  编码 (页面/组件)
  17:00 - 18:00  自测 + 提交 PR + 更新进度

Git 工作流:
  main ← feature/xxx (每个任务一个分支)
  每天至少一次 push
  PR 合并前必须: lint 通过 + typecheck 通过 + 手动预览测试

代码规范:
  - 组件文件: PascalCase.tsx
  - 工具函数: kebab-case.ts
  - 类型优先放在 src/types/
  - 所有 UI 文本中文，代码注释中文
  - 使用 Prettier 格式化（配置见 package.json）
```
