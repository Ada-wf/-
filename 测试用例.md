# 求职城市网 — 测试用例

> **版本**: v1.0
> **创建日期**: 2026-07-06
> **测试框架**: Vitest + Testing Library (单元/组件) | Playwright (E2E)
> **测试环境**: Node.js 20+ + Chromium (headless)

---

## 一、测试体系总览

```
层级                    工具                    范围               执行频率
─────────────────────────────────────────────────────────────────────────
L0  类型检查            tsc --noEmit            全量                每次 commit
L1  内容校验            自定义脚本               所有 .md/.yaml      每次 commit
L2  单元测试            Vitest                  工具函数/API 逻辑   每次 push
L3  组件测试            Vitest + Testing Lib     UI 组件             每次 push
L4  集成测试            Vitest                  API Route / 数据管线 每次 deploy 前
L5  E2E 测试            Playwright              完整用户流程         每次 deploy 前
L6  人工测试            -                       兼容性/视觉/AI 质量  每周
```

### TC-CONTENT-001: Markdown Frontmatter 格式校验

| 属性 | 内容 |
|------|------|
| **测试文件** | `scripts/validate-content.ts` |
| **被测对象** | `content/careers/` 下所有 `.md` 文件 |
| **前置条件** | 至少存在 1 个 `.md` 文件 |

```
用例 1: 合法文件 — 所有必填字段完整
  Given: software-engineer.md 包含 slug, name, tagline, categoryPath(discipline/school/major), conceptImage, overview, responsibilities, suggestion, tags
  When:  运行 validate-content.ts
  Then:  输出 "✅ software-engineer.md — 通过"，exit code = 0

用例 2: 缺少必填字段 slug
  Given: test-career.md 的 frontmatter 缺少 slug
  When:  运行 validate-content.ts
  Then:  输出 "❌ test-career.md — 缺少必填字段: slug"，exit code ≠ 0

用例 3: categoryPath 嵌套不完整
  Given: test-career.md 的 categoryPath 只有 discipline，缺 school 和 major
  When:  运行 validate-content.ts
  Then:  输出 "❌ test-career.md — categoryPath.school 缺失"

用例 4: slug 格式非法
  Given: test-career.md 的 slug = "软件工程师" (含中文)
  When:  运行 validate-content.ts
  Then:  输出 "❌ test-career.md — slug 格式非法 (只允许小写字母/数字/连字符)"

用例 5: tags 为空数组
  Given: test-career.md 的 tags = []
  When:  运行 validate-content.ts
  Then:  输出 "⚠️ test-career.md — tags 为空 (建议至少 3 个标签)"

用例 6: educationRequired 值非法
  Given: test-career.md 的 educationRequired = "博士后" (不在枚举中)
  When:  运行 validate-content.ts
  Then:  输出 "❌ test-career.md — educationRequired 值非法: 博士后"

用例 7: industryOutlook 值非法
  Given: test-career.md 的 industryOutlook = "excellent"
  When:  运行 validate-content.ts
  Then:  输出 "❌ test-career.md — industryOutlook 值非法: excellent"

用例 8: relatedCareers 引用不存在的 slug
  Given: test-career.md 的 relatedCareers 包含 "non-existent-career"
  When:  运行 validate-content.ts
  Then:  输出 "⚠️ test-career.md — relatedCareers 引用了不存在的职业: non-existent-career"

用例 9: salaryRange source 字段缺失
  Given: test-career.md 有 salaryRange.junior/senior 但缺 salaryRange.source
  When:  运行 validate-content.ts
  Then:  输出 "⚠️ test-career.md — salaryRange.source 缺失 (建议标注数据来源)"
```

### TC-CONTENT-002: YAML 分类数据完整性校验

| 属性 | 内容 |
|------|------|
| **测试文件** | `scripts/validate-content.ts` |
| **被测对象** | `content/categories/tsinghua-categories.yaml` |

```

  Given: tsinghua-categories.yaml
  When:  检验 disciplines 数组长度
  Then:  disciplines.length === 12

用例 2: 门类 ID 唯一性
  Given: tsinghua-categories.yaml
  When:  检验 disciplines 中所有 id
  Then:  无重复 id

用例 3: 学院 disciplineId 引用有效
  Given: tsinghua-categories.yaml
  When:  检验每个 school.disciplineId 是否存在于 disciplines[].id
  Then:  所有 school.disciplineId 有效

用例 4: 专业 schoolId 引用有效
  Given: tsinghua-categories.yaml
  When:  检验每个 major.schoolId 是否存在于对应 discipline 的 schools[].id

  Given: tsinghua-categories.yaml
  When:  检验每个 major.careerSlugs
  Then:  所有 major.careerSlugs.length > 0

用例 6: careerSlugs 与实际 .md 文件一致性
  Given: tsinghua-categories.yaml + content/careers/ 下所有 .md
  When:  交叉校验 YAML 中提到的 careerSlugs 和 .md 中的 slug
  Then:  YAML 中引用的 slug 有对应的 .md 文件；.md 中标记的 major 也被 YAML 引用
```
